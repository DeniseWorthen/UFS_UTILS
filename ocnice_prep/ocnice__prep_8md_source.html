3 # Introduction
4 
5 The ocnice_prep program will downscale a MOM6 or CICE6 1/4deg tripole restart file to a desired lower resolution tripole warm-start file using ESMF regridding.

13 ## Creating a warm-start file for MOM6 or CICE6 from a restart file at higher resolution
14 
15 A warmstart file can be created from an existing restart file (at higher resolution) by using ESMF Regridding to map fields
16 to a destination grid. For MOM6, the procedure produces a warm-start file (composed of T,S,U,V and Interface Height), where the
17 utility creates the interface heights (``eta``) using the sea surface height (``sfc``) and interface thickness (``h``) in
18 the MOM6 restart. A full restart file is not generated because MOM6 has resolution dependent parameterizations, so that restart
19 files for different resolutions might contain different fields. For CICE6, a file consisting of all the restart fields is produced.
20 While in the form of a "true" restart, it is also more appropriately considered to be a warm-start file.
21 
22 For MOM6, the following MOM_input settings can then be used with the warm-start file (e.g. ``ocean.mx100.nc``):
23 
24 ```
25 ! === module MOM_state_initialization ===
26 INIT_LAYERS_FROM_Z_FILE = False ! [Boolean] default = False
27  ! If true, initialize the layer thicknesses, temperatures, and salinities from a
28  ! Z-space file on a latitude-longitude grid.
29 ! WARMSTARTS
30 THICKNESS_CONFIG = "file" ! default = "uniform"
31  ! A string that determines how the initial layer thicknesses are specified for a
32  ! new run:
33  ! file - read interface heights from the file specified
34  ! by (THICKNESS_FILE).
35 THICKNESS_FILE = "ocean.mx100.nc" !
36  ! The name of the thickness file.
37 ADJUST_THICKNESS = True ! [Boolean] default = False
38  ! If true, all mass below the bottom removed if the topography is shallower than
39  ! the thickness input file would indicate.
40 THICKNESS_TOLERANCE = 0.1 ! [m] default = 0.1
41  ! A parameter that controls the tolerance when adjusting the thickness to fit
42  ! the bathymetry. Used when ADJUST_THICKNESS=True.
43 INTERFACE_IC_VAR = "eta" ! default = "eta"
44  ! The variable name for initial conditions for interface heights relative to
45  ! mean sea level, positive upward unless otherwise rescaled.
46 INTERFACE_IC_RESCALE = 1.0 ! [various] default = 1.0
47  ! A factor by which to rescale the initial interface heights to convert them to
48  ! units of m or correct sign conventions to positive upward.
49 TS_CONFIG = "file" !
50  ! A string that determines how the initial temperatures and salinities are
51  ! specified for a new run:
52  ! file - read velocities from the file specified
53 TS_FILE = "ocean.mx100.nc" !
54  ! The initial condition file for temperature.
55 TEMP_IC_VAR = "Temp" ! default = "PTEMP"
56  ! The initial condition variable for potential temperature.
57 SALT_IC_VAR = "Salt" ! default = "SALT"
58  ! The initial condition variable for salinity.
59 SALT_FILE = "ocean.mx100.nc" ! default = "ocean.mx100.nc"
60  ! The initial condition file for salinity.
61 VELOCITY_CONFIG = "file" ! default = "zero"
62  ! A string that determines how the initial velocities are specified for a new
63  ! run:
64  ! file - read velocities from the file specified
65  ! by (VELOCITY_FILE).
66 VELOCITY_FILE = "ocean.mx100.nc" !
67  ! The name of the velocity initial condition file.
68 U_IC_VAR = "u" ! default = "u"
69  ! The initial condition variable for zonal velocity in VELOCITY_FILE.
70 V_IC_VAR = "v" ! default = "v"
71 ```

73 For CICE6, the warm-start file (e.g. ``ice.mx100.nc``) can be used directly in the ``ice_in`` namelist:
74 
75 ```
76  runtype = 'initial'
77  runid = 'unknown'
78  ice_ic = 'ice.mx100.nc'
79 ```
80 
81 The remapping is done using an ESMF remapping via a RouteHandle, where the both the source and destination mask values are ``0``.
82 This ensures that only water points are mapped between the grids. Extrapolation is used to map destination points which are unmapped
83 because the source grid was not a water point. Since the "land mask" varies with depth, the utility makes use of dynamic masking if
84 required. This allows the same RouteHandle to be used for all depths, with masking at each depth done "on the fly".
85 
86 In addition, mapping for MOM6 is done bilinearly, whereas for CICE6 it is done using 'nearest-source-to-destination' mapping. This
87 ensures that the thermodynamic fields in the CICE6 restart, which are for each vertical layer (typically 7) and each thickness
88 category (typically 5), are consistent when mapped.
89 
90 Fields in the restart files for both MOM6 and CICE6 are defined at locations on the Arakawa C-grid. Similar to the ``cpld_gridgen``
91 utility, these are referred to here as the ``Ct``, ``Cu``, ``Cv`` and ``Bu`` locations. Vectors in MOM6 are natively located at
92 the north and east faces (``Cv`` and ``Cu``) while CICE vectors are located at ``Bu`` locations.
93 
94 The general procedure, for either MOM6 or CICE6 is as follows:
95 
96 1. Fields are retrieved from the source file and placed on the ``Ct`` grid location. For vector fields, the vectors are rotated
97 from their orientation along model dimensions (IJ) to eastward-northward (EN) directions.
98 2. Fields are packed into arrays by mapping type (bilinear or conservative) and dimensionality.
99 3. Fields are remapped using ESMF. This remapping may use dynamic Masking.
100 4. Vector fields are re-rotated back to model index direction (EN->IJ) and remapped back to their native stagger locations.
101 5. Fields are written into a warm-start file.
102 
103 The following files are required:
104 
105 - The ESMF mesh file for the source and destination grids. These are available as products of the ``cpld_gridgen`` utility.
106 - The remapping weights for mapping fields to and from ``Ct`` grid locations on both the souce and destination grids. These are also available as products of the ``cpld_gridgen`` utility.
107 - A text file (csv) listing the fields to be remapped for either the ocean or ice restart file.
108 - A namelist file defining the type of regridding desired (``ocean`` or ``ice``) and the locations of the needed weights.
109 
110 Because the required fields for MOM6 are located in two separate restart files, a single file containing all the necessary fields
111 must be generated for the ``ocean`` case. This is done using NetCDF operators (NCO) with the following commands, assuming that the
112 two required MOM6 restart files are available locally and are named ``MOM.res.nc`` and ``MOM.res_1.nc``.
113 
114 ```
115 ncks -v Temp,Salt,h,u MOM.res.nc ocean.nc
116 ncks -v v,sfc -A MOM.res_1.nc ocean.nc
117 ```
